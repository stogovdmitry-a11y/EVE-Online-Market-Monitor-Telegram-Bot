# Инструкция по локальному запуску EVE Market & Industry Monitor

Эта инструкция поможет вам запустить приложение локально на вашем компьютере (Windows) так, чтобы оно:
1. Запускалось автоматически в фоновом режиме при входе в систему (без висящих окон терминала).
2. Имело удобные ярлыки на рабочем столе для быстрого запуска и остановки.

Мы рассмотрим два простых способа: **Способ 1 (через простые скрипты VBS/BAT, без установки дополнительных утилит)** и **Способ 2 (профессиональный, через PM2)**.

---

## Предварительные требования (для всех способов)

1. **Установите Node.js** (версии 18 или выше), если он еще не установлен.
   * [Скачать Node.js для Windows](https://nodejs.org/)
2. **Скачайте файлы приложения** (экспортируйте ZIP-архив из меню настроек AI Studio или клонируйте репозиторий) и распакуйте их в любую удобную папку на компьютере (например, `C:\EVE-Monitor`).
3. **Установите зависимости и соберите проект**:
   Откройте терминал (командную строку) в папке с проектом и выполните:
   ```bash
   npm install
   npm run build
   ```
4. **Настройте файл `.env`**:
   Создайте файл с именем `.env` в корневой папке проекта (рядом с `package.json`) со следующим содержимым:
   ```env
   PORT=3000
   NODE_ENV=production
   EVE_CLIENT_ID=ваш_client_id_с_developers_eveonline_com
   EVE_CLIENT_SECRET=ваш_client_secret
   # URL вашего локального приложения (обязательно укажите http://localhost:3000)
   APP_URL=http://localhost:3000
   ```

## Способ 1. Запуск через тихие скрипты (VBS/BAT) — Без стороннего софта

Этот метод идеален, так как использует встроенные инструменты Windows. Мы создадим невидимый скрипт запуска, который скроет черное окно консоли.

### Шаг 1. Создание скрипта запуска в фоне (`start-hidden.vbs`)
Создайте в корне папки проекта файл `start-hidden.vbs` со следующим содержимым:
```vbs
Set WshShell = CreateObject("WScript.Shell")
' Запускает start-app.bat в скрытом режиме (0 = скрыть окно)
WshShell.Run Chr(34) & WshShell.CurrentDirectory & "\start-app.bat" & Chr(34), 0, False
Set WshShell = Nothing
```

### Шаг 2. Создание основного скрипта запуска (`start-app.bat`)
Создайте файл `start-app.bat` в корне папки проекта:
```batch
@echo off
cd /d "%~dp0"
:: Проверяем, собрано ли приложение, и запускаем его
if not exist "dist\server.cjs" (
    npm run build
)
node dist/server.cjs
```

### Шаг 3. Создание скрипта остановки (`stop-app.bat`)
Создайте файл `stop-app.bat` в корне папки проекта:
```batch
@echo off
:: Мягко завершаем процесс Node.js, который слушает порт 3000
for /f "tokens=5" %%a in ('netstat -aon ^| findstr :3000 ^| findstr LISTENING') do (
    taskkill /f /pid %%a
    echo Приложение на порту 3000 успешно остановлено.
    goto end
)
echo Приложение не запущено или порт 3000 свободен.
:end
pause
```

### Шаг 4. Настройка автозапуска при входе в систему
Чтобы приложение загружалось при входе в Windows:
1. Нажмите комбинацию клавиш `Win + R`.
2. Введите `shell:startup` и нажмите Enter. Откроется папка Автозагрузки Windows.
3. Нажмите правой кнопкой мыши по файлу `start-hidden.vbs` в вашей папке проекта, выберите **Копировать**.
4. Перейдите в открывшуюся папку автозагрузки, нажмите правой кнопкой мыши в пустом месте и выберите **Вставить ярлык**.
*Теперь при каждом включении компьютера и входе в систему бот будет незаметно запускаться в фоновом режиме.*

### Шаг 5. Создание удобных ярлыков на Рабочем столе
Вы можете создать ярлыки для быстрого запуска и остановки прямо на рабочем столе:
1. **Ярлык Запуска**: Нажмите правой кнопкой мыши на `start-hidden.vbs` -> **Отправить** -> **Рабочий стол (создать ярлык)**. Переименуйте его на рабочем столе в `Запустить EVE Монитор`.
2. **Ярлык Остановки**: Нажмите правой кнопкой мыши на `stop-app.bat` -> **Отправить** -> **Рабочий стол (создать ярлык)**. Переименуйте его в `Остановить EVE Монитор`.
   * *Совет:* Чтобы ярлык остановки выглядел красиво, вы можете зайти в его Свойства -> Сменить значок и выбрать подходящую иконку (например, красную кнопку).

## Способ 2. Профессиональный запуск через менеджер процессов PM2

PM2 — это мощный инструмент для фонового запуска Node.js приложений. Он автоматически перезапускает приложение при сбоях и собирает логи.

### Шаг 1. Установка PM2 глобально
Откройте консоль и установите PM2:
```bash
npm install -g pm2
```

### Шаг 2. Создание конфигурации запуска (`ecosystem.config.js`)
Создайте в корне проекта файл `ecosystem.config.js`:
```javascript
module.exports = {
  apps: [
    {
      name: 'eve-industry-monitor',
      script: './dist/server.cjs',
      env: {
        NODE_ENV: 'production',
        PORT: 3000
      },
      autorestart: true,
      watch: false,
      max_memory_restart: '200M'
    }
  ]
};
```

### Шаг 3. Создание ярлыков для PM2

1. **Ярлык «Запустить EVE Монитор»** (`start.bat`):
   Создайте файл `start.bat` в корне проекта:
   ```batch
   @echo off
   cd /d "%~dp0"
   pm2 start ecosystem.config.js
   ```
   Создайте на него ярлык на Рабочем столе. Окно терминала откроется на секунду, запустит фоновый процесс и сразу закроется.

2. **Ярлык «Остановить EVE Монитор»** (`stop.bat`):
   Создайте файл `stop.bat` в корне проекта:
   ```batch
   @echo off
   cd /d "%~dp0"
   pm2 stop eve-industry-monitor
   ```
   Создайте на него ярлык на Рабочем столе.

3. **Логи и статус**:
   Вы всегда можете проверить, работает ли приложение, введя команду `pm2 status` или посмотреть логи ошибок командой `pm2 logs`.

### Шаг 4. Настройка автозапуска PM2 в Windows
Для автоматического запуска PM2 при старте системы:
1. Установите специальный сервис автозапуска:
   ```bash
   npm install -g pm2-windows-startup
   pm2-startup install
   ```
2. Сохраните текущие запущенные процессы в список автозапуска:
   ```bash
   pm2 start ecosystem.config.js
   pm2 save
   ```
