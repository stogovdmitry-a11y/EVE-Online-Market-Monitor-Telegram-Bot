# EVE Online Market Monitor Bot 🚀

EVE Online Market Monitor Bot — это полнофункциональное приложение (веб-интерфейс + Telegram-бот) для автоматического мониторинга торговых ордеров (Buy & Sell) в EVE Online. Бот в реальном времени отслеживает ваши ордера в регионе **The Forge** (включая торговый хаб **Jita IV - Moon 4**) и мгновенно отправляет уведомления в Telegram, если ваши цены перебивают конкуренты.

---

## ✨ Основной функционал

1. **Мониторинг ордеров в реальном времени**: Автоматический опрос ESI API EVE Online для получения активных ордеров ваших персонажей.
2. **Анализ конкуренции (Undercut Detection)**: Сравнение цен ваших ордеров с актуальным стаканом публичного рынка региона The Forge. Бот вычисляет, лидирует ли ваша цена или её перебили.
3. **Мгновенные уведомления в Telegram**: Оповещения с детальной информацией (название товара, ваша цена, лучшая цена на рынке и разница в ISK) отправляются прямо в ваш Telegram-чат.
4. **Интеграция с EVE Online SSO**: Безопасная авторизация реальных игровых персонажей через официальный протокол OAuth 2.0 (Eve Developer Portal).
5. **Симулятор виртуальных персонажей**: Встроенная система симуляции персонажей и ордеров для тестирования работы алгоритмов и Telegram-оповещений без необходимости авторизации реальных аккаунтов.
6. **Панель управления (Dashboard)**:
   - Список подключенных персонажей и детальный статус каждого ордера.
   - Удобный конфигуратор Telegram-бота (статус работы, ввод токена, логирование).
   - Терминал логов в реальном времени для отслеживания системных событий.
   - Встроенный интерактивный симулятор Telegram-чата для тестирования команд прямо из веб-интерфейса.

## 🤖 Команды Telegram-бота

Вы можете взаимодействовать с ботом в Telegram с помощью следующих команд:
* `/start` — Инициализация бота, приветствие и краткая инструкция по привязке персонажей.
* `/list` — Вывод списка привязанных к чату персонажей, их активности, а также **списка всех перебитых на данный момент ордеров** с разницей цен.
* `/check` — Запуск принудительной мгновенной проверки всех ордеров на рынке и отправка свежих уведомлений о перебитых позициях.

## 🛠️ Файлы конфигурации и хранение ключей

Для работы приложения требуются API-ключи и настройки авторизации. Они настраиваются в следующих местах:

### 1. Переменные окружения (`.env`)
Для локальной разработки скопируйте файл `.env.example` в `.env` и заполните параметры:
```env
# Токен вашего Telegram-бота, полученный от @BotFather
TELEGRAM_BOT_TOKEN="123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ"

# Параметры вашего приложения на портале EVE Online Developers
EVE_CLIENT_ID="ваш_client_id_с_eve_developers"
EVE_CLIENT_SECRET="ваш_client_secret_с_eve_developers"

# URL-адрес вашего запущенного приложения (для перенаправления OAuth)
APP_URL="http://localhost:3000"
```
*При развертывании в облаке (например, в Google Cloud Run через AI Studio) данные переменные настраиваются в панели **Secrets / Environment Variables**.*

### 2. Локальная база данных (`db.json`)
Вся операционная информация приложения сохраняется локально в файле `db.json` в корне проекта. Здесь хранятся:
* Данные авторизованных персонажей и их зашифрованные OAuth-токены (Access & Refresh tokens) для работы с ESI API.
* Привязанные Telegram `chatId` пользователей.
* Настройки работы бота (активность поллинга, текущий токен).
* История системных и рыночных логов.
* История отслеживаемых ордеров и их прошлые статусы (для предотвращения дублирования уведомлений).

## 🚀 Пошаговый процесс настройки и запуска

### Шаг 1. Создание приложения EVE Online SSO
1. Перейдите на портал [EVE Online Developers](https://developers.eveonline.com/) и авторизуйтесь под своей учетной записью.
2. Нажмите **Create New Application**.
3. Заполните название и описание приложения.
4. В поле **Connection Type** выберите **Authentication & API Access**.
5. Добавьте в список разрешений (**Scopes**) следующие пункты:
   - `esi-markets.structure_markets.v1`
   - `esi-assets.read_assets.v1`
   - `esi-characters.read_orders.v1`
6. В поле **Callback URL** укажите адрес вашего приложения с эндпоинтом обратного вызова:
   - Для локального запуска: `http://localhost:3000/api/auth/callback`
   - Для продакшена: `https://<ваш_домен_приложения>/api/auth/callback`
7. Сохраните приложение и скопируйте полученные **Client ID** и **Secret Key**.

### Шаг 2. Создание Telegram-бота
1. Найдите бота `@BotFather` в Telegram.
2. Отправьте команду `/newbot` и следуйте инструкциям для создания нового бота.
3. Скопируйте полученный **HTTP API Token** (Telegram Bot Token).

### Шаг 3. Запуск приложения

#### Локальный запуск:
1. Установите зависимости проекта:
   ```bash
   npm install
   ```
2. Создайте файл `.env` в корневой директории и укажите все скопированные на предыдущих шагах ключи.
3. Запустите сервер в режиме разработки:
   ```bash
   npm run dev
   ```
4. Откройте в браузере `http://localhost:3000`.

#### Использование в Telegram:
1. Перейдите по ссылке на вашего Telegram-бота или найдите его по имени.
2. Нажмите кнопку **Запустить** (или отправьте `/start`).
3. На веб-панели управления (Dashboard) добавьте ваших персонажей (реальных через EVE SSO или тестовых виртуальных).
4. Запустите бота в панели управления (переключатель "Статус Telegram-бота").
5. Бот автоматически свяжет персонажей с вашим Telegram-аккаунтом при отправке команд в чат (например, `/check` или `/list`), и начнет присылать уведомления при перебитии ордеров!

## 📂 Структура проекта

* `server.ts` — Основной бэкенд-сервер на Express. Содержит логику авторизации EVE SSO, планировщик фоновых проверок рынка, обработчик лонг-поллинга Telegram API и эндпоинты для веб-интерфейса.
* `db.json` — Локальное файловое хранилище (БД).
* `src/` — Фронтенд-приложение на React + TypeScript (Vite):
  - `src/components/Dashboard.tsx` — Главный экран панели управления.
  - `src/components/BotConfig.tsx` — Настройки Telegram-бота и статус поллинга.
  - `src/components/OrdersGrid.tsx` — Сетка активных ордеров с цветовой индикацией статуса конкуренции.
  - `src/components/LogTerminal.tsx` — Визуальный терминал для мониторинга логов в реальном времени.
  - `src/components/ChatSimulator.tsx` — Интерактивный симулятор Telegram для ручного тестирования взаимодействия с ботом.
  - `src/components/CharactersList.tsx` — Список добавленных персонажей и интерфейс SSO авторизации.
